# 논문 계획 (로봇산업 연계안)
## 하네스 엔지니어링 기반 저비용 LLM의 스마트팩토리 구매 프로세스 적용

---

## 핵심 연구 질문 (Research Question)

> "스마트팩토리 환경의 로봇 부품 조달 프로세스에서,
> GPT-3.0급 저성능 LLM에 하네스 엔지니어링을 결합하면
> 고성능 모델 수준의 신뢰도를 더 낮은 비용으로 달성할 수 있는가?"

---

## 논문 제목 (안)

*"스마트팩토리 부품 조달의 지능화: 하네스 엔지니어링 기반 저비용 LLM의 신뢰도 최적화 설계"*

---

## 왜 로봇 산업인가 — 선택 근거

### 1. 구조적 유사성 (이론적 정당성)

하네스 엔지니어링의 개념은 로보틱스 제어 이론에서 유추할 수 있음.

```
[로보틱스]
결정론적 컨트롤러 ──제어──▶ 불확실한 물리 액추에이터 (센서 오차, 환경 변수)

[하네스 엔지니어링]
결정론적 하네스   ──제어──▶ 확률적 LLM (환각, 비결정적 출력)
```

두 구조 모두 **예측 불가능한 시스템을 결정론적 레이어로 통제**한다는 본질이 동일.
이 유추는 논문의 이론적 배경을 강화하는 근거로 활용.

### 2. 산업적 필요성 (경영적 정당성)

| 문제 | 내용 |
|---|---|
| 부품 조달 복잡도 | 산업용 로봇 1대당 수천 개의 부품 → 견적 비교 업무량 폭증 |
| 납기 리스크 | 부품 1개 지연이 전체 라인 스톱으로 이어짐 → 오류 비용 극대화 |
| 공급망 변동성 | 글로벌 공급망 이슈로 대체 공급업체 탐색 빈도 증가 |
| 전문 인력 부족 | 로봇 부품 전문 구매 인력 수급 어려움 |

### 3. RPA → LLM 전환 문제 (차별화 포인트)

```
기존 자동화 (RPA)
├─ 완전 결정론적, 룰 기반
├─ 정확하지만 예외 상황 처리 불가
└─ 공급망 변동 시 룰 재작성 필요

신규 시도 (LLM 단독)
├─ 유연하고 자연어 이해 가능
├─ 하지만 확률적 → 구매 오류 발생 가능
└─ 기업 투입 신뢰도 미달

해결 (LLM + 하네스)
├─ LLM의 유연성 유지
├─ 하네스가 RPA 수준의 결정론적 신뢰도 부여
└─ 공급망 변동에도 규칙 레이어만 업데이트하면 대응 가능
```

## 타깃 서브프로세스

스마트팩토리 맥락에서 가장 구조화된 2개에 집중.

| 서브프로세스 | 구체 내용 | 하네스 적용 포인트 |
|---|---|---|
| **로봇 부품 RFQ 처리** | 부품 번호(PN) 기반 견적 수집 및 비교 | 스펙 불일치 감지, 예산 상한 차단 |
| **대체 공급업체 평가** | 기존 공급사 납기 지연 시 대체사 스코어링 | 인증 요건 체크, 품질 기준 검증 |

## 하네스 아키텍처 (스마트팩토리 버전)

```
[입력: 부품 번호(PN), 요구 스펙, 납기 조건]
        ↓
 ① Input Structuring Layer
    - 부품 번호 표준화 (IEC/ISO 규격 매핑)
    - 필수 스펙 필드 (내압, 온도, 인증) 누락 시 반환
        ↓
 ② LLM (GPT-3.0급)
    - 복수 공급업체 견적 비교 요약 생성
    - 대체 부품 적합성 초안 평가
        ↓
 ③ Validation Loop  ←──────────────────┐
    - 스펙 허용 오차 범위 초과 여부 체크       │
    - 미인증 공급업체 필터링                 │
    - 납기 조건 미충족 시 LLM 재질의 (최대 N회)──┘
        ↓
 ④ Tool Constraint Gate
    - 단가 × 수량 > 승인 한도 → 자동 에스컬레이션
    - 공급망 블랙리스트 공급업체 자동 차단
    - 단일 공급사 의존도 임계치 초과 시 경고
        ↓
 ⑤ Audit Trail 생성
    - 부품별 의사결정 근거 로그 (품질 감사 대응)
    - ISO 9001 / IATF 16949 컴플라이언스 문서 자동 출력
        ↓
[출력: 구매 담당자에게 최종 추천 공급업체 + 근거 문서]
```

## 논문 구조

### 1. 서론
- 스마트팩토리 확산과 부품 조달의 복잡도 증가
- 기존 RPA의 한계와 LLM 도입 시도의 신뢰도 문제
- 하네스 엔지니어링을 통한 해결 방향 제시
- 연구 목적 및 논문 구성

### 2. 이론적 배경
- **로보틱스 제어 이론과의 구조적 유추**
  - 결정론적 컨트롤러 + 확률적 액추에이터 → 하네스 + LLM
- LLM의 확률적 특성과 환각(Hallucination) 문제
- 하네스 엔지니어링 개념 정의
  - Validation Loop / Self-Correction / Tool Constraint
- RPA와 LLM 에이전트의 비교 및 하네스의 위치
- 스마트팩토리 부품 조달 프로세스의 구조화 특성

### 3. 시스템 설계 (데이터사이언스 파트)
- 하네스 아키텍처 전체 플로우
- 로봇 부품 RFQ 처리 모듈 설계
- 대체 공급업체 스코어링 모듈 설계
- 검증 규칙(Validation Rules) 정의
  - 스펙 허용 오차 범위
  - 인증 요건 체크리스트
  - 단일 공급사 의존도 임계치

### 4. 비용-신뢰도 분석 (경영 파트)
- 시나리오 비교

| 시나리오 | API 비용 | 오류율 | 적합성 |
|---|---|---|---|
| 저성능 LLM 단독 | 낮음 | 높음 | 투입 불가 |
| **저성능 LLM + 하네스** | **낮음** | **낮음** | **최적** |
| 고성능 LLM 단독 | 높음 | 낮음 | 비용 과다 |

- 비용 시뮬레이션: `API 토큰 비용 × 평균 재질의 횟수`
- 납기 지연 리스크 감소에 따른 기대 손실 절감액 추정
- ROI 분석 (구현 비용 대비 오류 비용 절감)

### 5. 논의
- **경영적 시사점:** 공급망 거버넌스, 감사 가능성(ISO 대응), 구매 인력 역할 재정의
- **한계:** 신규 부품 번호 대응의 어려움, 초기 규칙 설계 비용, 하드웨어 연동 미포함
- **향후 연구:** 생산 라인 실시간 데이터와의 연동, 멀티에이전트 확장

### 6. 결론
