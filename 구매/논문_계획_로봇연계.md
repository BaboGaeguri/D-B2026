> 고성능 모델 수준의 신뢰도를 더 낮은 비용으로 달성할 수 있는가?"

## 왜 스마트팩토리 부품 조달인가

### 산업적 배경

스마트팩토리에서 생산 라인은 수천 종의 부품으로 구성되며, 단 하나의 부품 조달 오류가 전체 라인 가동 중단(Line Stop)으로 이어진다. 이 환경에서 구매 오류의 비용은 일반 사무 업무에 비해 압도적으로 크다. 동시에 글로벌 공급망 불안정으로 대체 공급업체 탐색 빈도가 증가하면서, 반복적인 고부하 구매 업무가 구조적으로 발생하고 있다.

| 문제 | 내용 |
|---|---|
| 업무량 | 산업용 로봇 1대당 수천 개 부품 → 견적 비교 업무 폭증 |
| 오류 비용 | 부품 1개 납기 지연 또는 스펙 불일치 → 라인 전체 스톱 |
| 공급망 변동성 | 대체 공급업체 탐색 빈도 증가 → 비정형 판단 업무 증가 |
| 인력 한계 | 전문 구매 인력으로 감당 불가한 처리량 |

### 기존 자동화의 한계와 LLM의 역할

현재 제조업 구매 자동화는 RPA(Robotic Process Automation)로 이루어져 있으나, 공급망 변동처럼 예외 상황이 발생하면 룰을 전부 재작성해야 한다. 이 한계를 극복하기 위해 LLM 도입이 시도되고 있으나, LLM의 확률적 출력 특성이 구매 오류를 유발해 실무 투입이 어렵다.

```
RPA (기존)
├─ 정확하지만 예외 상황 처리 불가
└─ 공급망 변동 시 룰 전면 재작성 필요

LLM 단독 도입 (시도)
├─ 유연하고 자연어 처리 가능
└─ 확률적 출력 → 스펙 오류, 예산 초과 등 구매 실수 발생

LLM + 하네스 (본 논문 제안)
├─ LLM의 유연성 유지
├─ 하네스가 결정론적 검증으로 오류를 차단
└─ 공급망 변동 시 규칙 레이어만 업데이트하면 대응 가능
```

이것이 본 논문에서 스마트팩토리 부품 조달을 선택한 핵심 이유다. **오류 비용이 크고, 기존 자동화의 한계가 명확하며, LLM+하네스 구조가 그 공백을 메울 수 있다.**

## 타깃 서브프로세스

구매 프로세스 전체가 아닌 아래 2개에 집중.

| 서브프로세스 | 구체 내용 | 하네스 적용 포인트 |
|---|---|---|
| **RFQ 처리** | 부품 번호(PN) 기반 복수 견적 수집 및 비교 분석 | 스펙 불일치 감지, 예산 상한 차단 |
| **대체 공급업체 평가** | 납기 지연 발생 시 대체 공급업체 스코어링 | 인증 요건 체크, 품질 기준 검증 |

## 하네스 아키텍처

```
[입력: 부품 번호(PN), 요구 스펙, 납기 조건, 예산]
        ↓
 ① Input Structuring Layer
    - 비정형 견적서 → 정형 JSON 변환
    - 필수 필드 (스펙, 납기, 단가, 인증번호) 누락 시 즉시 반환
        ↓
 ② LLM (GPT-3.0급)
    - 복수 공급업체 견적 비교 요약 생성
    - 대체 공급업체 적합성 초안 평가
        ↓
 ③ Validation Loop  ←─────────────────┐
    - 스펙 허용 오차 범위 초과 여부 체크      │
    - 미인증 공급업체 필터링                │
    - 예산 상한선 초과 여부 체크            │
    - 위 항목 실패 시 LLM 재질의 (최대 N회) ─┘
        ↓
 ④ Tool Constraint Gate
    - 단가 × 수량 > 승인 한도 → 자동 에스컬레이션
    - 블랙리스트 공급업체 자동 차단
    - 단일 공급사 의존도 임계치 초과 시 경고
        ↓
 ⑤ Audit Trail 생성
    - 의사결정 근거 전체 로그화
    - ISO 9001 컴플라이언스 문서 자동 출력
        ↓
[출력: 구매 담당자에게 최종 추천 공급업체 + 근거 문서]
```

## 논문 구조

### 1. 서론
- 스마트팩토리 확산과 부품 조달 업무의 복잡도 증가
- RPA의 구조적 한계와 LLM 단독 도입의 신뢰도 문제
- 하네스 엔지니어링을 통한 해결 방향
- 연구 목적 및 논문 구성

### 2. 이론적 배경
- LLM의 확률적 특성과 환각(Hallucination) 문제
- 하네스 엔지니어링 개념 정의
  - Validation Loop: LLM 출력을 규칙 기반으로 검증하고 재질의하는 루프
  - Self-Correction: 검증 실패 시 LLM 스스로 출력을 수정하는 메커니즘
  - Tool Constraint: 에이전트의 도구 사용 권한을 사전에 제한하는 구조
- RPA와 LLM 에이전트 비교 및 하네스의 위치
- 스마트팩토리 부품 조달 프로세스의 구조화 특성과 AI 적합성

### 3. 시스템 설계 (데이터사이언스 파트)
- 하네스 아키텍처 전체 플로우
- RFQ 처리 모듈 상세 설계
- 대체 공급업체 스코어링 모듈 상세 설계
- Validation Rules 정의
  - 스펙 허용 오차 기준
  - 인증 요건 체크리스트
  - 단일 공급사 의존도 임계치

### 4. 비용-신뢰도 분석 (경영 파트)

#### 시나리오 비교

| 시나리오 | API 비용 | 오류율 | 실무 투입 가능성 |
|---|---|---|---|
| 저성능 LLM 단독 | 낮음 | 높음 | 불가 |
| **저성능 LLM + 하네스** | **낮음** | **낮음** | **가능** |
| 고성능 LLM 단독 | 높음 | 낮음 | 가능하나 비용 과다 |

- 비용 시뮬레이션: `API 토큰 비용 × 평균 재질의 횟수`
- 라인 스톱 방지에 따른 기대 손실 절감액 추정
- ROI 분석: 하네스 구현 비용 대비 오류 비용 절감

### 5. 논의

- **경영적 시사점:** 공급망 거버넌스 강화, ISO 감사 대응 자동화, 구매 인력의 역할 재정의
- **한계:** 신규 부품 번호 대응의 어려움, 초기 검증 규칙 설계 비용
- **향후 연구:** 생산 라인 실시간 데이터 연동, 멀티에이전트 확장
