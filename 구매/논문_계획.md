# 논문 계획: 하네스 엔지니어링 기반 저비용 LLM의 기업 구매 프로세스 적용

---

## 핵심 연구 질문 (Research Question)

> "GPT-3.0급 저성능 LLM은 단독으로는 기업 구매 프로세스에 투입 불가한 수준의 오류율을 보이지만,
> 하네스 엔지니어링을 통해 고성능 모델 대비 동등한 신뢰도를 더 낮은 비용으로 달성할 수 있는가?"

**핵심 주장:** 비용 vs. 신뢰도 트레이드오프를 하네스가 해결한다.

---

## 논문 제목 (안)

*"하네스 엔지니어링 기반 저비용 LLM의 기업 구매 프로세스 적용: 비용 효율성과 운영 신뢰도의 최적화 설계"*

---

## 타깃 서브프로세스

구매 프로세스 전체가 아닌 아래 2개에 집중.

| 서브프로세스 | 하네스 적용 포인트 | 선택 이유 |
|---|---|---|
| **RFQ 처리** (견적 요청서 분석 및 비교) | Validation Loop + Tool Constraint | 입력/출력이 정형화, 오류 감지 기준 명확 |
| **공급업체 평가 스코어링** | Self-Correction + Audit Trail | 평가 기준이 체크리스트 형태 → 검증 가능 |

> 계약서 초안은 비정형 텍스트 비중이 높아 "향후 연구"로 분류

---

## 하네스 아키텍처

```
[입력: 견적서 PDF / 공급업체 데이터]
        ↓
 ① Input Structuring Layer
    - 비정형 문서 → 정형 JSON 변환
    - 필수 필드 누락 시 즉시 반환
        ↓
 ② LLM (GPT-3.0급)
    - 견적 비교 요약 생성
    - 공급업체 평가 점수 초안 생성
        ↓
 ③ Validation Loop  ←──────────────┐
    - 예산 상한선 초과 여부 체크       │
    - 필수 항목(납기, 보증조건) 존재 여부 │
    - 실패 시 LLM에 재질의 (최대 N회)  ──┘
        ↓
 ④ Tool Constraint Gate
    - 승인 한도 초과 시 자동 에스컬레이션
    - 블랙리스트 공급업체 자동 차단
        ↓
 ⑤ Audit Trail 생성
    - 모든 판단 근거 로그화
    - 컴플라이언스 문서 자동 출력
        ↓
[출력: 구매 담당자에게 최종 추천 + 근거 문서]
```

## 논문 구조

### 1. 서론
- 기업 AI 도입의 딜레마: 고성능 = 고비용, 저성능 = 저신뢰
- 구매 부서의 반복 업무 현황 (선행연구 or 통계)
- 연구 목적 및 논문 구성 안내

### 2. 이론적 배경
- LLM의 확률적 특성과 환각(Hallucination) 문제
- 하네스 엔지니어링 개념 정의
  - Validation Loop
  - Self-Correction
  - Tool Constraint
- 구매 프로세스의 구조화 특성과 AI 적합성

### 3. 시스템 설계 (데이터사이언스 파트)
- 하네스 아키텍처 전체 플로우
- RFQ 처리 모듈 설계
- 공급업체 평가 스코어링 모듈 설계
- 검증 기준(Validation Rules) 정의

### 4. 비용-신뢰도 분석 (경영 파트)
- 시나리오 비교

| 시나리오 | 비용 | 신뢰도 |
|---|---|---|
| 저성능 LLM 단독 | 낮음 | 낮음 |
| **저성능 LLM + 하네스** | **낮음** | **높음** |
| 고성능 LLM 단독 | 높음 | 높음 |

- 비용 시뮬레이션: API 호출 비용 × 재질의 횟수
- 기대 ROI 및 OPEX 절감 추정

### 5. 논의
- **경영적 시사점:** 거버넌스, 감사 가능성, 조직 변화 관리
- **한계:** 완전 비정형 업무 적용 불가, 초기 규칙 설계 비용
- **향후 연구:** 계약서 검토, 멀티에이전트 확장

### 6. 결론

## 데이터 및 검증 방법

실험 데이터 확보가 어려우므로 **시뮬레이션 기반 검증**으로 진행.

1. **데이터 소스:** 공공데이터포털, 나라장터 공개 조달 데이터로 테스트셋 구성
2. **모델:** GPT-3.5-turbo를 저성능 모델 프록시로 사용해 실제 API 호출
3. **측정 지표:**
   - 오류율: 하네스 적용 전/후 비교
   - 비용: 토큰 수 × 단가 × 재질의 횟수
4. **결과 제시:** 저성능 단독 vs. 저성능+하네스 vs. 고성능 단독 3-way 비교

---

## 경영적 정당성 근거

- 대기업 구매팀의 반복 업무 비중: 전체의 60~70% 이상
- KPI 명확: 구매단가 절감률, 처리 시간(리드타임), 오류율
- 저성능 모델 사용의 당위성: 구조화된 양식 내에서만 작동하면 충분
