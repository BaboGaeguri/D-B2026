# 논문 계획 (전체 상장사 사업보고서 리스크 분석)

## Harness Engineering 기반 저비용 LLM의 비용-신뢰도 최적화 설계

### — 상장사 사업보고서 리스크 감지를 사례로 —

---

## 문제 의식

최근 LLM 모델의 등장으로 증권사·회계법인·기업 IR팀 등에서 사업보고서 분석 자동화를 시도하고 있다. 그러나 고성능 AI 모델일수록 API 비용이 높고, 저성능 모델은 오류율이 높아 실무 투입이 어렵다는 구조적 딜레마가 존재한다.

> "기업이 비싼 최고급 모델만 쓰지 않고도 Harness Engineering을 통해 신뢰성을 확보할 수 있는가?"

---

## 가설

**H1.** 기업이 고비용·고성능의 최신 AI 모델만 쓰지 않고도 Harness Engineering을 통해 동등한 리스크 감지 신뢰도를 확보할 수 있다.

**H2.** 저비용 AI에 Harness Engineering을 적용함으로써 비용 구조를 개선하고 더 높은 ROI를 산출할 수 있다.

---

## 왜 사업보고서 리스크 분석인가

### 산업적 배경

| 문제 | 내용 |
| --- | --- |
| 문서 규모 | 전체 상장사 2,500개+, 사업보고서 1개당 수백 페이지 |
| 반복성 | 매년 동일 구조로 반복 제출 → 자동화 적합 |
| 오류 비용 | 리스크 미감지 → 투자 손실, 감사 실패, 규제 위반 |
| **정답 존재** | **감사의견·수정공시가 공식 오류 기록으로 공개** |

### 왜 이 주제가 평가 가능한가

다른 문서 분석 주제와 달리 사업보고서는 두 가지 외부 공식 정답이 존재한다.

```text
정답 1: 감사의견 (4-class 분류)
  적정 / 한정 / 부적정 / 의견거절
  → 외부 감사인이 독립적으로 판단한 공식 기록 (DART 공개)

정답 2: 수정공시 발생 여부 (이진 분류)
  발생(1) / 미발생(0)
  → 기업이 공식으로 오류를 인정한 기록 (DART 공개)
```

LLM + 하네스의 판단 결과를 이 두 정답과 직접 비교해 F1-score로 객관적 측정이 가능하다.

## 연구 설계

### 독립 변수

| 변수 | 수준 |
| --- | --- |
| **Harness 수준** | Group A ~ D (4단계) |
| **LLM 모델** | 저성능 (GPT-3.5-turbo) / 고성능 (GPT-4o) |

### 4그룹 실험 설계

```text
Group A: LLM Only
  → 하네스 없이 LLM 단독 판단
  → 기준선(Baseline) 역할

Group B: LLM + Validation
  → LLM 출력에 재무 수치 정합성 검증만 추가
  → 자산 = 부채 + 자본, 전기 대비 이상값 체크

Group C: LLM + Validation + Constraint
  → B에 Tool Constraint Gate 추가
  → K-IFRS 기준 위반 항목 자동 차단
  → 감사의견 모순 시 재질의 강제

Group D: Full Harness
  → C에 수정공시 이력 교차검증 + Audit Trail 추가
  → 완전한 하네스 구조
```

각 그룹을 **저성능 LLM / 고성능 LLM** 두 모델로 각각 실행 → 총 8개 조합 비교

### 평가 지표

| 지표 | 측정 방법 |
| --- | --- |
| **정확도** | 감사의견 예측 F1-score |
| **오류율** | 수정공시 미감지율 (False Negative Rate) |
| **비용** | API 토큰 비용 × 평균 재질의 횟수 |
| **소요 시간** | 문서 1건당 평균 처리 시간 (초) |

## 하네스 아키텍처

```text
[입력: DART 사업보고서 텍스트 + 재무제표 수치]
        ↓
 ① Input Structuring Layer
    - 사업보고서 → 섹션별 정형 JSON 변환
    - 리스크 요인·재무제표·주석 섹션 분리
        ↓
 ② LLM (GPT-3.5 or GPT-4o)
    - 리스크 항목 추출 및 위험 등급 판단
    - 감사의견 예측 (적정/한정/부적정/의견거절)
        ↓
 ③ Validation (Group B부터 적용)
    - 재무 수치 내부 정합성 체크
      (자산 = 부채 + 자본 ±1%)
    - 전기 대비 주요 수치 이상값 감지 (±30% 초과)
        ↓
 ④ Constraint Gate (Group C부터 적용) ←────────┐
    - K-IFRS 기준 위반 항목 자동 플래그           │
    - 감사의견 모순 시 LLM 재질의 강제            │
    - 재질의 횟수 상한 설정 (최대 N회) ────────────┘
        ↓
 ⑤ Audit Trail + 수정공시 이력 (Group D만 적용)
    - 과거 수정공시 이력과 현재 판단 교차검증
    - 판단 근거·재질의 횟수 전체 로그화
        ↓
[출력: 리스크 등급 + 감사의견 예측 + 근거 요약]
```

## 예상 결과: 비용-신뢰도 곡선

핵심 연구 결과는 아래 형태의 그래프로 표현된다.

```text
신뢰도(F1)
  ↑
1.0|                    ● D (저성능+Full Harness)
   |              ● C (저성능+Validation+Constraint)
   |         ● B (저성능+Validation)
0.5|    ● A (저성능 단독)
   |
   |         ● A' (고성능 단독)  ← 비용 대비 D와 유사 수준
   |
   └──────────────────────────────→ 비용(원/건)
        낮음                   높음
```

**기대 발견:**

- 하네스 수준이 높아질수록 신뢰도 상승, 비용은 소폭 증가
- Full Harness 저성능 LLM이 하네스 없는 고성능 LLM과 비슷한 신뢰도 달성
- ROI 관점에서 저성능 + Full Harness가 최적점

## 논문 구조

### 1. 서론

- 사업보고서 리스크 분석의 경영적 중요성과 수동 검토의 한계
- 고성능 LLM 단독 도입의 비용 문제와 저성능 LLM의 신뢰도 문제
- Harness Engineering을 통한 비용-신뢰도 최적화 가능성 제시

### 2. 이론적 배경 및 선행 연구

- LLM의 확률적 특성과 환각(Hallucination) 문제
- Harness Engineering 개념 정의
  - Validation Loop: LLM 출력을 규칙 기반으로 검증하고 재질의하는 루프
  - Self-Correction: 검증 실패 시 LLM 스스로 출력을 수정하는 메커니즘
  - Tool Constraint: 에이전트의 도구 사용 권한을 사전에 제한하는 구조
- 선행 연구: preprints202604.0428 (Agent Harness Survey) 주요 발견 인용
- DART 공시 제도 및 감사의견 분류 체계

### 3. 연구 방법 (데이터사이언스 파트)

- 4그룹 × 2모델 실험 설계 상세
- DART API 데이터 수집 파이프라인
- 하네스 아키텍처 각 레이어 상세 설계
- Validation Rules 정의

  | 검증 규칙 | 기준 | 정답 출처 |
  | --- | --- | --- |
  | 재무 수치 정합성 | 자산 = 부채 + 자본 ±1% | K-IFRS |
  | 전기 대비 이상값 | 주요 수치 ±30% 초과 | 통계 기준 |
  | 감사의견 모순 | LLM 예측 vs 실제 의견 | DART 감사보고서 |
  | 수정공시 이력 | 과거 수정공시 발생 여부 | DART 수정공시 |

### 4. 결과 분석 (경영 파트)

#### 4.1 그룹별 비용-정확도 비교

| 그룹 | 모델 | 정확도 (F1) | 비용 (원/건) | 소요 시간 |
| --- | --- | --- | --- | --- |
| A | 저성능 | 측정값 | 측정값 | 측정값 |
| B | 저성능 | 측정값 | 측정값 | 측정값 |
| C | 저성능 | 측정값 | 측정값 | 측정값 |
| D | 저성능 | 측정값 | 측정값 | 측정값 |
| A | 고성능 | 측정값 | 측정값 | 측정값 |

#### 4.2 ROI 분석

```text
ROI = (리스크 미감지 비용 절감액 - 하네스 구현 비용) / 하네스 구현 비용

리스크 미감지 비용:
  수정공시 1건 평균 주가 하락률 × 시가총액 = 기대 손실
  → 그룹별 오류율 차이 × 기대 손실 = 절감액 추정
```

#### 4.3 업종별 하네스 효과 비교

제조업 / IT·소프트웨어 / 금융 / 서비스 4개 업종 간 하네스 효과 차이 분석

### 5. 논의

- **경영적 시사점:** 전문 애널리스트 없이 1차 리스크 스크리닝 자동화 가능
- **한계:** 사업보고서 텍스트 길이로 인한 컨텍스트 한계, 업종별 회계 특수성
- **향후 연구:** 실시간 공시 모니터링 연동, 멀티에이전트 확장

### 6. 결론

Harness Engineering은 기업 AI 시스템의 비용-신뢰도 곡선을 상방 이동시킨다.
저비용 모델에 적절한 하네스를 설계함으로써 고성능 모델 수준의 신뢰도를 낮은 비용으로 달성할 수 있다.
